1. Перевод — главная проблема
Бесплатный Google Translate (gtx endpoint) — нелегален для коммерческого использования, нестабилен, без SLA. Варианты:

Google Cloud Translation API (платный, $20/1M символов)
DeepL API ($5.49/1M символов, лучше качество для EU языков)
LLM (Claude/GPT) — лучшее качество для контекстных переводов, дороже
Собственная модель на базе open-source (MarianMT, NLLB) — долго, но $0 за inference
2. Качество перевода
Сейчас paragraph-level — это минимум. Для коммерческого продукта нужна:

Терминология по отраслям (compliance, финансы, юридика)
Glossary — пользователь загружает свой словарь терминов, они не переводятся или переводятся строго по словарю
Post-editing UI — предпросмотр перевода с возможностью правки до скачивания
Оценка качества — подсветка "подозрительных" переводов
3. Аутентификация и мультитенантность

Регистрация/логин (email + OAuth)
Лимиты: бесплатный тариф (N страниц/месяц), платный
Изоляция данных — каждый пользователь видит только свои файлы
API-ключи для интеграций
4. Оплата

Stripe/Kaspi/CloudPayments интеграция
Тарифные планы: по объёму (символы/страницы), по подписке
Учёт потребления (метринг)
5. Инфраструктура

Очередь задач (Redis/BullMQ) вместо in-memory jobs Map — сейчас при рестарте всё теряется
S3/MinIO для хранения файлов вместо локальных папок
Rate limiting на API
HTTPS, CORS, CSP headers
Мониторинг (uptime, ошибки API переводчика, метрики по задачам)
6. Юридическое

Terms of Service, Privacy Policy
GDPR/PDPA — удаление данных пользователя, retention period для файлов
Не хранить документы дольше необходимого
7. Расширение форматов

PDF (самый востребованный, самый сложный)
ODP/ODS/ODT (LibreOffice)
Subtitle файлы (.srt, .vtt)
